  • AYSBased: ()

Body

Media type: application/json

Type: object

Properties
  • name: required (string - maxLength: 15)

    Name for the bridge to be created. The name can't be "default".

  • hwaddr: (string)

    MAC address for the bridge to be created. If none, a random MAC address will be assigned.

  • networkMode: required (one of none, static, dnsmasq)

    Networking mode

  • nat: required (boolean)

    If true, SNAT will be enabled on this bridge.

  • setting: required (object)

    Networking settings are depending on the selected 'networkMode' as follows:

    • networkMode = 'none': No settings Bridge won't get any IP settings.
    • networkMode = 'static': settings={'cidr': 'ip/net'} Bridge will get assigned the given IP address.
    • networkMode= 'dnsmasq': settings={'cidr': 'ip/net', 'start': 'ip', 'end': 'ip'} Bridge will get assigned the IP address specified for 'cidr' and each running container that is attached to this IP address will get an IP address within the specified start/end range. Netmask of the range is the netmask part of the provided CIDR. If 'nat' is set to true, SNAT rules will be automatically added in the firewall.
    • cidr: required (string)
    • start: required (string)
    • end: required (string)

Headers

    URL to the created bridge resource

Body

Media type: application/json

Type: Run

HTTP status code 409

Body

Type: object

Properties

    name of the vdisk. need to be unique

  • size: required (integer)

    Size of the vdisk in GiB

  • blocksize: required (integer - default: 4096)

    blocksize in bytes

  • type: required (one of boot, db, cache, tmp)
    • boot: A boot disk is a redundant disk that can be initialized based on a templatevdisk. See templatevdisk property. Boot disks should only be used for the operating system and applications installed on the operating system. Performance of bootdisks is not optimized because that defeats their purpose.
    • db: A db disk is a redundant disk optimized for performant block storage. DB disks should be used for running reliable high io intensive workloads such as databases, key value stores, ...
    • cache: A cache disk is a persistent non redundant disk optimized for performant block storage. Cache disks should be used for high io intensive workloads which don't need to be highly reliable.
    • tmp: A tmp disk is a non persistent non redundant disk optimized for performant block storage. Content in tmp disks is only available while mounted. Tmp disks should be used for swap and mounts in /tmp.
